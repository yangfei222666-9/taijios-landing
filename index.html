The expensive failure

Most agent stacks do not die. They pretend to work.

真正危险的不是报错，而是旧产物还在、自动化没跑、学习层失败，却被页面包装成“最新成功”。TaijiOS 专门处理这种运营层故障。

01

Stale latest

预检失败后只刷新错误文件，旧 digest 和 action sheet 还留在 latest 目录，下游误以为刚刚成功。

02

Silent downgrade

主任务成功、学习层失败，如果没有 degraded 状态，就会被吞成 ok，后面复盘只能翻日志猜。

03

Bad channel as good

provider、DNS、relay、quota 没验证就当可用，整个系统会在关键路径上制造假进展。

Operating loop

One workflow. Five hard gates.

TaijiOS 不靠“感觉系统在跑”。每一步都有输入、输出、失败原因和最终仲裁，能停机，也能恢复。

01 / PREFLIGHT

先验通道

DNS、provider、quota、model name、ritual artifact 先过门禁。坏通道不准进入主流程。

02 / RUN

执行主链

digest、brain_feed、CV、promote 各自输出 run artifact，不靠聊天记录当系统状态。

03 / EVENT

事件流留痕

每轮输入、输出、异常、仲裁写入事件流。没有事件流的成功，视为未验证成功。

04 / REVIEW

多轮复查

DeepSeek、Doubao、Haiku 与规则仲裁分层，主判断和对抗审不混在一起。

05 / PROMOTE

晋升或隔离

候选先入学习层，连续命中才可晋升。degraded 永远不能直接进入 judgment 链。

默认输出：verdict / blocker / next action / risk 默认边界：learning_only before judgment
Quality chain

多模型不是堆数量，是分工。

每个模型承担不同职责。主判断、中文语境、对抗审、规则仲裁分开，才能减少自我合理化。

R1 to R4

质量链的目标不是省钱，而是防漂移。relay 可以补位，但关键流程优先走可验证的原生 provider。额度不足、429、鉴权失败必须 loud 报告。

R1
DeepSeek v4 flash主判断与快速复查，优先发现工程层问题。
R2
Doubao中文语境、叙事和易经学习层复核。
R3
Anthropic Haiku对抗审，专门找循环论证、框架漂移和隐藏假设。
R4
Rules确定性仲裁：ok / degraded / failed，不让模型自己给自己盖章。

Public stance

我们不把学习层包装成裁决层。

TaijiOS 可以吸收易经、事件样本和多模型意见，但这些都必须先进入 learning_only。没有连续运行、事件流和晋升兑现前，不夸大“自进化”。

Why it matters

AI 系统的护城河不是更多按钮，是可复盘样本。

30 天后仍然成立的东西，才是硬实力。TaijiOS 把 daily ritual、cross validation、promote algorithm 和 quant brain_feed 统一成可追踪事件流，减少“今天热闹、明天找不到证据”的损耗。

  • Audit每次运行有状态、有证据、有阻塞。
  • Recover失败能停，恢复后能按同一命令链重跑。
  • Improve命中样本进入候选池，过门槛才晋升。
Operating principles

TaijiOS 的规则很少，但很硬。

这些规则不是口号，是工作流的退出条件和审计标准。

不假装成功

失败、未验证、阻塞、假活必须明确指出。

不把坏通道当可用

服务没验证、自动化没心跳、数据没落地，都不算完成。

先收口，再扩张

每次只补最小闭环，不用新工具掩盖旧阻塞。

输出必须可执行

给顺序、动作、检查点和风险，不给空话。

Start with one chain

如果你的 AI 工作流已经在跑，先做一次 silent-failure audit。

不用先重构。先把当前链路的预检、失败语义、latest artifact、事件流和恢复命令查清楚。能看见问题，系统才有资格继续进化。
